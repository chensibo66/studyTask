# 面向对象基础



### 一.面向对象概述

```
面向对象并不是一个技术,而是一种编程思想.强调以什么形式写代码,以什么思路解决问题.
```



### 二.类和对象

```
1.类和对象
	什么是类?
		是一类具有共同属性和行为事物的描述,可以理解为设计图
	什么是对象?
		对象是类的实体,是看得见摸得着的实体
2.类的定义
	类的组成
		1.属性: 成员变量(类种方法外的变量)
		2.行为: 成员方法(不加static修饰的方法)
3.对象的使用
	创建对象使用哪个关键字?
		使用new关键字
	对象创建完毕直接打印属性什么结果?	
		会打印属性对应数据类型的默认初始值,引用类型为null 整数为0 小数为0.0
```



### 三.对象内存图

```
1.Java内存分配
	栈内存特点?
		方法被调用后,进入栈内存,方法中的变量(局部变量)会随着方法进入栈内存. 当方法调用结束,会从			栈内存消失,此时方法中的变量(局部变量)也会消失
	堆内存特点?
		new出来的内容进堆内存,成员变量跟随对象进入堆内存. 进堆就会开辟空间,产生地址值,该地址在使		用完毕后,在系统空闲时间会被Java垃圾回收器清理
2.对象内存(单个对象):新new的对象存放在堆中,并产生一个地址值赋给在栈中的临时变量,当代码中需要使用到					对象中的变量或方法时,先通过栈中的临时变量找到堆中的相关地址,并通过此地址访问					  堆中内容
3.对象内存(多个对象):与单个对象接近,只是在堆中多开辟了多出来对象的空间并赋给栈中新增临时变量
4.对象内存(多个对象指向相同):栈中不同的变量指向同个堆地址,共用同些内容,通过哪个变量调用都一样
5.成员变量和局部变量
	成员变量和局部变量的区别?
		1.类中位置: 成员变量在类中方法外, 局部变量在方法中或者方法声明上
		2.内存中位值: 成员变量在堆内存存储, 局部变量在栈内存存储
		3.生命周期: 成员变量随着对象的创建而存在,随着对象的消失而消失, 局部变量随着方法的调用而			存在,随着方法的弹栈而消失
		4.初始化值: 成员变量进入堆内存,有默认初始化值, 局部变量没有,需要先赋值才能使用
```

### 四.封装

```
1.private关键字
	private关键字的语义和作用?
		1.是一个权限修饰符
		2.可以修饰成员(成员变量和成员方法)
		3.被private修饰的成员只能在本类中访问
	被private修饰的成员外界如何使用呢?
    	我们要提供对应的被public修饰的getXxx和setXxx方法
2.this关键字
	this关键字的语义和作用?
		1.this代表本类对象的引用(地址)
		2.谁调用方法,方法中的this就代表谁
		3.我们使用this来解决局部变量和成员变量重名问题
3.封装
	面向对象三大特征?
		封装,继承,多态
		封装: 将代码的具体实现细节隐藏在类中,不让外界随便访问
		优点: 提高了代码的安全性,复用性
	封装的体现?
		1.将代码抽取到方法中, 调用者直接使用方法名, 是对代码的封装
		2.将代码抽取到类中, 也是对代码的封装
		3.使用private私有成员变量, 这是对属性的封装, 外界如果使用我们需要提供对应的getXxx和			setXxx方法
    封装的好处?
    	1.提高代码的复用性(方法)
    	2.提高代码的安全性(私有)
```

### 五.构造方法

```
1.构造方法
	构造方法的作用及格式?
		本质作用是创建对象
		public 方法名(){}
2.构造方法的注意事项
	构造方法的使用注意?
		1.如果我们不提供构造,系统会给一个默认的空参构造
		2.如果我们给出了带参构造,那么系统就不会再提供空参构造了(这时用户就不能使用空参构造创建对			象)
		结论: 两种构造都提供,用户根据自己情况随意调用
3.JavaBean
	1.什么是JavaBean?
		就是Java中的一个类,用于封装数据,比如Student,Phone,Car
	2.一个标准的JavaBean包含什么?
		1.成员变量: 必须使用private修饰
		2.构造方法: 提供两种,空参带参都提供
		3.成员方法: 必须提供private修饰的成员变量所对应的getXxx和setXxx方法, 其他方法根据需求			  走
```
